Posted on 2021-05-24 Edited on 2021-06-21

AWS SAAのまとめ

EC2

  • インスタンス多数起動
    • ブートストラップ
    • ゴールデンイメージ
  • インスタンスのオプション
    • オンデマンドインスタンスのキャバシティー
      • saving plans
        • キャバシティーのオプション
  • タグ数の上限
    • 50個
  • リザーブドインスタンス
    • Standard RI
  • スポットインスタンス
    • スポットブロック
      　 - 6時間まで連続する処理が保証される
    • スポットフルート
  • リザーブドインスタンス
    • スタンダード
      • 同じリージョン内全てAZを変更できる
  • 完全なスナップショット
    　 - EC2の停止
汎用SSD プロビジョンドIOPS スループット ColdHDD
スループット/ボリューム 250mib/s 1000 mib/s 500 mib/s 250 mib/s

VPC

  • サブネット
    • 上限200個
  • VPNの連携
    • オンプレ側にカスタマイズゲートウェイ
    • AWS側に仮想プライベードゲートウェイ
  • エンドポイント
    • gateway
      • API gateway
      • cloudwatch
    • private link
      • Dynamo DB
      • S3
  • ENI
    • ホットアタッチ
      • 実行中のアタッチ
    • ウォームアタッチ
      • 停止中のアタッチ
    • コールドアタッチ
      • インスタンスが起動中のアタッチ
  • Transit Gateway
    • 中央ハブを介してVPCとオンプレと接続
  • Egress-Only インターネットゲートウェイ

    CloudTrail

  • アカウントの操作履歴

RDS

  • パフォーマンス改善
    • コストをかかるけど、効果が高い(高速クエリ)
      • Elastic Cache
  • コスト重視
    • レードレプリカ
  • 自動バックアップ
    • Oracleデータベースを起動して自動バックアップする
  • AutoScaling
    • RDSインスタンスの追加ではなく容量の自動追加
  • レードレプリカ
    • リージョンごとレードレプリカ構成
    • オフロード
      • Selectクエリ（古いデータが必要となる）
  • プロキシ
    　 - アプリとRDSのコネクション管理
可用性 スループット レイテンシ
